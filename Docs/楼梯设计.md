# 楼梯设计

## 核心概念

楼梯不是链式连接，是电梯。

```
❌ 链式（错误模型）          ✅ 电梯（正确模型）
1F ─楼梯→ 2F ─楼梯→ 3F      1F ─┐
                              2F ─┼─ 楼梯井 ─ 任意互通
                              3F ─┘
```

## 楼层本质

每一层是一张独立的 Map，逻辑上平行。没有"上下"的物理关系，只有高程编号（elevation）区分。

```
Map(elev=1)    Map(elev=2)    Map(elev=3)
┌──────────┐  ┌──────────┐  ┌──────────┐
│          │  │          │  │          │
│   ┌──┐   │  │   ┌──┐   │  │   ┌──┐   │
│   │楼│   │  │   │楼│   │  │   │楼│   │
│   │梯│   │  │   │梯│   │  │   │梯│   │
│   └──┘   │  │   └──┘   │  │   └──┘   │
│          │  │          │  │          │
└──────────┘  └──────────┘  └──────────┘
     ↕              ↕              ↕
     └──────── 同一位置 ────────────┘
              = 楼梯井
```

## 楼梯井（Stair Shaft）

多层楼梯在不同 Map 上占据相同 IntVec3 位置 → 形成楼梯井。

楼梯井内任意两层直通，不需要经过中间层。

判定规则：
- `Building_Stairs` 在 Map A 的位置 `(x, z)`
- Map B 的 `(x, z)` 也有 `Building_Stairs`
- → A 和 B 通过此楼梯井直通

```csharp
// FloorMapUtility.FindStairsToFloor
// 在当前 Map 找到一个楼梯，其位置在目标楼层也有楼梯
Building_Stairs FindStairsToFloor(Pawn pawn, Map currentMap, int targetElevation)
{
    foreach stairs on currentMap:
        targetMap = GetMapForElevation(targetElevation)
        if HasStairsAtPosition(targetMap, stairs.Position):
            if pawn.CanReach(stairs):
                return stairs  // 这个楼梯井通往目标层
}
```

## 传送机制

pawn 到达楼梯 → `StairTransferUtility.TransferPawn` → DeSpawn + Spawn 到目标层同位置。

```csharp
TransferPawn(pawn, stairs, targetElevation):
    destMap = GetMapForElevation(targetElevation)
    destPos = stairs.Position  // 同位置
    pawn.DeSpawn()
    GenSpawn.Spawn(pawn, destPos, destMap)
```

不走中间层。1F 直达 5F，一次传送。

## Job 中的高程编码

UseStairs job 用 `targetB` 存目标高程：

```csharp
job.targetB = new IntVec3(targetElevation, 0, 0);
```

JobDriver 读取：

```csharp
int TargetElevation => job.targetB.IsValid ? job.targetB.Cell.x : Stairs.targetElevation;
```

HaulToStairs / HaulAcrossLevel 用 `targetC`：

```csharp
job.targetC = new IntVec3(targetElevation, 0, 0);
```

## 可达楼层查询

```csharp
// FloorMapUtility.GetReachableFloors
// 返回通过此楼梯井可达的所有楼层
List<int> GetReachableFloors(Building_Stairs stairs):
    result = []
    foreach map in AllFloorMaps:
        if map != stairs.Map:
            if HasStairsAtPosition(map, stairs.Position):
                result.Add(GetMapElevation(map))
    return result
```

## 玩家交互

右键楼梯 → 浮动菜单列出所有可达楼层：

```
前往 1F
前往 2F  ← 当前层，灰色
前往 3F
前往 4F
```

选择后创建 UseStairs job，targetB = 选择的高程。

## 不做的事

1. ❌ 不要求楼梯有 `targetElevation` 属性指向特定楼层 — 楼梯井是动态发现的
2. ❌ 不逐层跳转（1F→2F→3F）— 直达
3. ❌ 不要求所有楼层都有楼梯 — 没楼梯的楼层就是不可达
4. ❌ 不限制楼梯数量 — 一层可以有多个楼梯井，pawn 选最近的
